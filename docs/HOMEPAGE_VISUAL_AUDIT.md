# Homepage Visual UX Audit

Last updated: 2026-05-21

## 2026 Civic UX Rubric

Score is 1-10. Target is 9.6 or higher per section.

- Task success, 25%: residents can complete common jobs quickly.
- Visual hierarchy, 20%: scan path is clear in 3-5 seconds.
- Mobile efficiency, 15%: low scroll cost, clear touch targets, no cramped actions.
- Accessibility and inclusion, 15%: plain language, older users, low vision, emergency users, and mobile visitors.
- Trust and civic credibility, 10%: official, calm, accurate, accountable.
- Content economy, 10%: every word, card, and link earns its space.
- 2026 polish, 5%: modern, responsive, restrained, and not gimmicky.

## Improvements Made

- Reduced mobile hero height and removed duplicate mobile emergency content from the hero.
- Kept the hero task set focused on the most common resident jobs.
- Tightened service card spacing and hid secondary descriptions on mobile.
- Removed homepage share controls from compact news cards.
- Tightened the emergency strip for mobile urgency.
- Reduced decorative transition space between emergency content and the community map.
- Shortened community, story, tourism, and about sections while preserving links into deeper pages.
- Added a compact Where Tennessee Began tourism bridge to the homepage using the existing tourism app screenshot and visitor-specific CTAs.

## Next Potential Gains

- Replace the SVG community map with a dedicated mobile list-first variant if user testing shows map taps are hard on older phones.
- Add analytics on top-task clicks, search queries, tourism-app clicks, and emergency-resource clicks.
- Use production Search.gov or analytics data to reorder hero and service tasks seasonally.
